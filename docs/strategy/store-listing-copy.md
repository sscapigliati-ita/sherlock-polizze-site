## Strategia A/B test (Store Listing Experiments)

In Play Console → Grow → Store listing experiments. Suggerite due esperimenti **non simultanei** (Play permette 1 esperimento attivo per asset):

### Esperimento 1 — Short description
- Variante A (controllo): testo attuale
- Variante B: la "principal" qui sopra ("Polizza che non capisci o sinistro respinto?...")
- Audience: 50/50, country IT
- Durata minima: 14 giorni o 100 installations per variante (Play decide)
- Metrica primaria: **Install rate** (visit → install)

### Esperimento 2 — Hero screenshot (dopo che 1 ha verdetto)
- Variante A: primo screenshot attuale (UI generica dell'app)
- Variante B: hero "Polizza che non capisci? Sinistro respinto?" (vedi `public/store-mockups.html`)
- Stessa setup
- Metrica primaria: install rate

**Non testare due cose insieme**: l'A/B test Play funziona su un asset per volta. Inoltre tieni un controllo per variazioni stagionali.
